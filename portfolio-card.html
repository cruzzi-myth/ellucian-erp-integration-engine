Key Reliability Patterns

Three architectural decisions that directly drove the system's reliability numbers — each solving a specific failure mode observed in production.

📤
Transactional Outbox + Idempotency

Before any third-party API call, the intent is written to an OutboxMessages table inside the same database transaction as the application state change. A background processor dispatches with a deterministic idempotency key — guaranteeing exactly-once delivery even across crashes.

~0% duplicate rate (was 3–5% before)
⏳
Calibrated Exponential Retry Ladder

Five retry attempts with delays calibrated against real failure patterns across 12 third-party APIs. 5s covers network blips; 30m covers DR events. Terminal failures bypass the ladder entirely and route straight to DLQ with full diagnostic context attached.

5s → 30s → 2m → 10m → 30m → DLQ
🔑
Runtime Multi-Tenant Isolation

Every institution is a fully isolated tenant. A TenantResolverMiddleware resolves the institution's API credentials, endpoint overrides, feature flags, and retry policies at runtime from a cached config store. Onboarding a new university is a data operation — no deployment required.

200+ tenants, zero shared state

// Writes intent atomically with the caller's DB transaction. // If the transaction rolls back, the outbox record rolls back too. public async Task<OutboxMessage> PublishAsync( string tenantId, string adapterName, string operationType, object payload, Guid sourceTransactionId) { // Deterministic key — same transaction always produces same key. // Re-processing a recovered outbox record is safe — downstream rejects duplicate. var idempotencyKey = DeriveKey(sourceTransactionId, operationType); var msg = new OutboxMessage { TenantId = tenantId, AdapterName = adapterName, IdempotencyKey = idempotencyKey, Status = OutboxStatus.Pending, NextRetryAt = DateTime.UtcNow }; _db.OutboxMessages.Add(msg); // Caller SaveChanges inside their transaction — atomicity guaranteed. return msg; }
// Core contract every third-party integration must fulfill. // The orchestrator only knows this interface — never the concrete system. public interface IIntegrationAdapter { string AdapterName { get; } Task<AdapterResult> ExecuteAsync( IntegrationRequest request, TenantContext tenantContext, CancellationToken ct = default); } // Adding a new integration = implement this + register it. // Retry, DLQ routing, tracing, tenant isolation — all automatic. public sealed class FinancialAidAdapter : IIntegrationAdapter { public string AdapterName => "financial-aid-platform"; public async Task<AdapterResult> ExecuteAsync( IntegrationRequest request, TenantContext tenant, CancellationToken ct) { // tenant.ApiCredentials resolved at runtime from Key Vault var response = await _httpClient.PostAsync(...); return AdapterResult.Success(); } }
// Demo ladder (production: 5s/30s/2m/10m/30m) public static readonly int[] DelaysSeconds = [3, 8, 15, 25, 45]; // ONE attempt per call. Delays stored as NextRetryAt in DB — // no thread.Sleep, non-blocking, survives process restarts. if (result.IsSuccess) { msg.Status = OutboxStatus.Delivered; return; } if (result.IsTerminalFailure || attempt >= MaxAttempts) { msg.Status = OutboxStatus.DeadLettered; await _dlq.RouteAsync(msg, result); // alert fires in 90s return; } // Schedule next attempt — background processor picks it up msg.NextRetryAt = DateTime.UtcNow.AddSeconds(DelaysSeconds[attempt - 1]); msg.Status = OutboxStatus.Pending;
